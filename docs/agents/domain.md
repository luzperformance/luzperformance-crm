# Domain docs

This repo is **single-context**.

## Read before exploring or changing work

1. `CONTEXT.md` at the repo root.
2. Relevant ADRs under `docs/adr/`.

## How engineering skills should use them

- Use `CONTEXT.md` for repo vocabulary and domain boundaries.
- Use `docs/adr/` for past decisions that constrain new work.
- If an ADR conflicts with the proposed change, call it out explicitly.

## File layout

- Root context: `CONTEXT.md`
- Repo-wide ADRs: `docs/adr/`
